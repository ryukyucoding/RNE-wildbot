# WildBot 實作方法介紹

## 一、物件抓取任務實作方法介紹

我們的物件抓取任務（Bear Mission）採用 **YOLO 物件偵測**結合 **ROS 2 Nav2 導航與視覺伺服（Visual Servoing）** 技術來完成自動化夾取與放置。

1. **自動導航與避障**：啟動任務時，機器人會先記錄初始的 Home 座標。接著系統利用 AMCL 進行定位，並透過 Nav2 產生全域路徑，同時我們實作了**自定義的局部避障機制（Obstacle Guard）**，能夠依據 LiDAR 扇區與深度相機數值動態減速或緊急煞車。
2. **目標尋找與追蹤**：系統訂閱 YOLO 節點的影像分析結果。若目標丟失，機器人會進入自適應搜尋狀態（如：原地旋轉搜尋或依據最後記憶的相對位置短暫前進）。
3. **視覺伺服逼近（Visual Servoing）**：鎖定熊玩偶後，控制迴圈根據 YOLO 輸出的水平像素誤差與深度，利用 PID 控制即時調整車體旋轉與前進速度，使車身精準對齊目標。
4. **夾取與返航**：當深度達到夾取觸發距離內，機器人煞停，啟動機械臂的 Inverse Kinematics（IK）進行三維空間抓取。夾取成功後，自動透過 Nav2 導航回起點（Home）並放下物件。如果 Nav2 回程遇到死角卡住（Stuck），系統有內建防呆的 Fallback 機制（後退重試或切換至純旋轉尋路）。

## 二、上橋取物任務實作方法介紹

上橋任務結合了 **SLAM 即時定位**與**自訂狀態機時間控制**，以應對橋面斜坡打滑與視線死角的問題。

1. **環境建圖與參數設定**：首先進行場景 SLAM 建圖，量測橋頭（Point A）與橋尾（Point B）的座標，並計算出橋梁的中軸線角度（Heading）。
2. **精準導航與對齊**：任務啟動後，使用 Nav2 導航至橋頭的精確座標。到達橋頭後，機器人會進行原地旋轉，將車身與橋梁中軸線完美對齊，確保不會偏離橋面。
3. **分段式過橋狀態機**：為了克服上坡打滑與下坡暴衝，我們實作了自訂 Action Server 進行分段控制：
   - **上坡段（CROSS_UP）**：設定較大動力全速前進（FORWARD）並維持特定秒數，確保能克服重力與摩擦力爬升。
   - **平臺段（CROSS_PLATFORM）**：轉為慢速前進，準備夾取橋面上的物件。
   - **下坡段（CROSS_DOWN）**：以慢速穩定下坡，避免暴衝翻車。
4. **持續定位**：過橋期間不切換至純 AMCL，而是持續運行 SLAM 以避免上坡時 LiDAR 掃不到特徵物造成的定位丟失。

## 三、開門任務實作方法介紹

開門任務採用了與物件抓取相似的視覺對齊框架，設計了「狀態機 (FSM)」來依序執行各項精密動作，並透過自訂的 YOLO 濾波條件克服環境雜訊。

1. **門把辨識與抗雜訊濾波**：使用 YOLO 模型 (`detection.pt`) 偵測門把（Knob）。為了解決場地邊緣招牌上的粗黑字體被誤判為目標的問題，我們在 YOLO 推論後處理加入了 **長寬比過濾（Aspect Ratio Filter）**。由於招牌文字的 Bounding Box 多為細長條狀，而門把較方正，系統預設只保留長寬比在 0.3 到 3.0 之間的偵測結果，完美排除了環境文字干擾。
2. **搜尋與精準視覺伺服 (Visual Servo Approach)**：
   - **導航與搜尋**：抵達門前導航點後，車體會原地緩慢旋轉搜尋門把。
   - **逼近**：鎖定門把後，透過 PID 運算控制左右輪速（Visual Servoing），讓門把始終保持在畫面中線。當 LiDAR 或 YOLO 深度低於安全停車距離（如 0.3m）時停車。
   - **精對齊（Fine Alignment）**：停車後進行原地的純旋轉微調，直到門把的水平像素誤差進入極小容差範圍內，確保手臂在 X 軸上完全正對門把。
3. **下壓開門操作**：
   - **手臂定位（ARM_AIM）**：打開夾爪，利用車身與門把已知距離，加上攝影機到手臂基座的結構補償值（Offset），直接將手臂精準移動到門把正上方 5cm 處。
   - **下壓（PRESS_DOWN）**：夾爪收合夾住/框住門把後，直接將手臂在 Z 軸下壓固定距離（如 7cm）。
   - **推開房門（OPEN_DOOR）**：下壓完成後，車體維持下壓姿態並啟動低速前進或後退（依據門的開向），運用車體動力將門順利推開，最後釋放手臂並重置姿態。

## 四、硬體改裝介紹(optional)

無硬體改裝。本團隊採用標準 Wildbot 設備與感測器。我們透過精準的軟體參數調整（包含車體機構中心到相機、手臂基座的 Offset 測量），在不改裝硬體的前提下，達到極高的操作精準度。

## 五、其他特殊設計(optional)

1. **Docker 模組化微服務架構**：
   我們將系統拆分為 `pros_app`（核心通訊與 SLAM）、`pros_car`（控制狀態機）與 `ros2_yolo_integration`（視覺辨識）等多個獨立 Docker 容器。此設計避免了 ROS 2 套件衝突，並允許團隊成員平行開發與測試不同模組。
2. **Digital Twin 虛實整合開發**：
   在部署至實體機器人前，大量使用了 `pros_twin` (Unity) 進行數位孿生模擬。這讓我們能夠在 macOS 本機環境安全地測試各種導航邏輯、狀態機轉換、以及 YOLO 辨識邏輯的串接，大幅減少實車測試的風險與時間成本。
3. **動態參數調整 (Dynamic Parameters)**：
   將所有環境變數、PID 常數、YOLO 過濾條件等皆開放為 ROS Parameters，能夠在不重新編譯的情況下，依據實車測試的當下環境與光線動態調整行為。
